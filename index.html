Overview

This group project required us to choose a startup from Y-Combinator, interpret their startup idea, and create a mockup of a mobile or web app for the startup. We chose the startup Agapé.

Agapé is a relationship wellness app for couples that nurtures both physical and emotional intimacy. Couples start their day with a daily question designed to spark meaningful conversation.

The goal of the project were to become familiar with the iterative design process assignment and to design an interface mockup for this startup idea.

Initial Sketches

First, we created intial sketches based on the description of the startup. We weren't allowed to look at the startup's actual designs, nor could we look at our teammates designs until we we're all finished. This way, we wouldn't be influenced by the initial design.

Conclusions

Based on each of our three sketches, we decided to include these following features in our low fidelity prototype:

  • A splash page that prompts the user to begin using the app.
  • A login page to simulate signing into an account.
  • Some sort of question/messaging page that allows users to converse about a chosen question.
  • A profile page that allows users to edit their account details.

Peer Critique

Based on our wireframes, we used Figma to create an initial prototype. As part of the design process, we held a studio session to recieve peer feedback on this design. Below is a summary of that feedback.

  • Maybe make the text on the Home Screen cover a little less of the width of the screen;
  • I think a little padding on the outside of the text would look good On the profile page, consider making each category (age, partner, etc. ) a different color or font style (e.g. bold) than the responses, as right now it’s a little hard to read
  • Small detail, but make sure to center the name and photo on the profile page
  • A little confused about what the purpose of app is
  • More prototyping after the “questions page” would be helpful
  • Calendar for streak isn't very interactive or useful unless you're searching for that information.
  • Probably not a priority to show unless a user clicks “streak” on profile

User Testing

Choosing a Task

Time to test the prototype! In order to conduct user tests on our prototype, we needed to decide on an overarching task for the users to complete. Here's the task we gave to our users:

Try to view your messages from November 2nd by using the calendar streaks!

  1. Click "get started"
  2. Click "Login"
  3. You should see the question that was asked that day, and the conversation you and your partner had as a result
